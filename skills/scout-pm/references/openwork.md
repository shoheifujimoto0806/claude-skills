# OpenWork 専門エージェント

OpenWork（旧Vorkers）は口コミサイトとしての文化が強く、「働きがい」「成長環境」「風通し」といったカルチャー面を重視するユーザーが多い。スカウトメッセージでもこの文化に合わせ、共感ベースのアプローチが効果的。

**重要な制約：** OpenWorkではスカウト文面にURLを記載するとBAN（アカウント停止）のリスクがある。URLは絶対に含めない。

## 検索条件の生成ルール

### OpenWork検索の特性
- 職種・業界・年収帯・勤務地などの**カテゴリ選択式**が中心
- フリーワード検索も可能だが、カテゴリ選択の方が精度が高い
- 口コミデータに基づく企業評価（総合評価、待遇面の満足度等）でのフィルタも可能な場合がある

### 広め条件

出力フォーマット：
```
■ OpenWork 広め条件

職種カテゴリ: ◯◯（OpenWorkの選択肢に合わせた表記）
業界: ◯◯
勤務地: ◯◯
年収帯: 指定なし
フリーワード: ◯◯（広めのキーワード）

想定ヒット感: 多め。ここから絞り込む前提。
```

### 絞り込み条件

出力フォーマット：
```
■ OpenWork 絞り込み条件

職種カテゴリ: ◯◯
業界: ◯◯
勤務地: ◯◯
年収帯: ◯◯万円〜◯◯万円
フリーワード: ◯◯（具体的なスキルキーワード）
経験年数: ◯年以上

想定ヒット感: 少なめだが精度高い。
```

### 検索条件のポイント
- OpenWorkの職種カテゴリ名をできるだけ正確に使う（例：「Webエンジニア」「営業・販売」等）
- フリーワードは日本語中心（OpenWorkは日本語プロフィールがほとんど）
- 年収帯は求人の提示年収よりも少し低めに設定すると母集団が広がる（年収アップ転職を狙う層を拾える）

## スカウト文面の生成ルール

### OpenWork向けの特性

| 項目 | ルール |
|------|-------|
| 日程調整URL | **含めない**（BAN対策） |
| トーン | やや丁寧・共感ベース。「働きがい」「成長環境」「裁量」を訴求 |
| 長さ | **300〜500文字** |
| フォーマット | プレーンテキスト（Markdown記法は使わない） |
| 言語 | 日本語 |

### 文面構成のポイント

OpenWorkユーザーは「口コミを見て企業を選ぶ」文化にいる。つまり、表面的な条件だけでなく「実際に中で働くとどうなのか」に関心が強い。

効果的なアプローチ：
1. 候補者の経験への敬意を示す（着目ポイント）
2. ポジションの「働きがい」「やりがい」を中心に訴求
3. 成長環境、裁量の大きさ、チーム文化に触れる
4. 条件面も記載するが、カルチャー面の後に置く
5. 返信を促す形で締める（URLなし）

### テンプレート

```
{候補者名}様

はじめまして。
合同会社Plus X代表の藤本と申します。
弊社は興味深いポジション・ビジネスを有している企業の紹介に強みを持つヘッドハンティング会社です。

{候補者名}様の御経歴を拝見し、ぜひともご紹介させていただきたいポジションがございます。

【着目させていただいたポイント】
{着目ポイント}

上記の御経験から、本ポジションにおいて大きな成果を発揮いただけると考えております。

【ポジションの魅力】
◎（働きがい・やりがいの観点で魅力を記載）
◎（成長環境・裁量の観点で魅力を記載）
◎（条件面の魅力を記載）

年収レンジ：◯◯万円〜◯◯万円
勤務地：東京都（リモート◯◯）

もし少しでも御興味をお持ちいただけましたら、求人のご案内のお時間を頂戴できないでしょうか。
ご返信にてご都合のよい日時をお知らせいただけましたら幸いです。
（所要時間：60分／オンライン）

御返信を心よりお待ちしております。
```

### 絶対に守ること
- **URLは絶対に含めない**（BAN対策）
- 日程調整はSpirIncのURLではなく、「ご返信にてご都合のよい日時をお知らせいただけましたら幸いです」の形式
- 企業名・プロダクト名は絶対に含めない
- 500文字を超えない

## ブラウザ操作ワークフロー（Claude in Chrome）

OpenWorkでの候補者リストアップをブラウザ上で行う場合の手順。
Claude in Chrome のツールを使って以下の順序で操作する。

**URL：** https://recruiting.vorkers.com/scout/candidates

### 手順

1. **候補者検索画面を開く**
   - 上記URLにアクセスする

2. **検索条件を入力して絞り込む**
   - 職種カテゴリ、業界、年収帯、フリーワードなどのフィルタを、このリファレンスで生成した検索条件に基づいて設定する
   - まず広め条件で母集団を確認し、多すぎれば絞り込み条件に切り替える

3. **ターゲットリストを新規作成する**
   - 検索結果を確認し、マッチ度が高い候補者をピックアップするための「ターゲットリスト」を新規作成する
   - リスト名は **求人のポジション名**（例：「PdM_テックフォワード案件」）で作成する

4. **候補者をターゲットリストに追加していく**
   - 検索結果の候補者一覧から、マッチ度が高そうな候補者をターゲットリストに追加する
   - 評価フレームワーク（マッチ度4以上）を参考に、スカウトを送る価値がある候補者を選定する
   - ターゲットリストに追加した候補者に対して、後からまとめてスカウトメッセージを送信できる
