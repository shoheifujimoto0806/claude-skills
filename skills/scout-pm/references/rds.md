# Recruit Direct Scout 専門エージェント

Recruit Direct Scout（リクルートダイレクトスカウト）はリクルートが運営するハイクラス転職サービス。転職意欲が比較的高い「顕在層」が多く、条件面（年収・ポジション・リモート可否）を具体的に提示することが効果的。

**重要な制約：** Recruit Direct ScoutではスカウトメッセージにURLを記載するとBAN（アカウント停止）のリスクがある。URLは絶対に含めない。

**自動化のポイント：** Recruit Direct Scoutには公式の「自動スカウト機能」があり、条件を設定しておくと自動でスカウトが送信される。この機能の活用も検索条件に含めて提案する。

## 検索条件の生成ルール

### Recruit Direct Scout検索の特性
- **フリーワード検索** + **業界・職種・年収帯などのカテゴリ選択**の併用
- カテゴリはリクルート独自の分類体系を使用
- 「レジュメ更新日」でフィルタすると、最近アクティブな候補者に絞れる

### 広め条件

出力フォーマット：
```
■ Recruit Direct Scout 広め条件

フリーワード: ◯◯（広めのキーワード）
業界: ◯◯（RDSのカテゴリ名に合わせた表記）
職種: ◯◯（RDSのカテゴリ名に合わせた表記）
年収帯: 指定なし
勤務地: 関東
レジュメ更新日: 直近3ヶ月以内（推奨）

想定ヒット感: 多め。ここから絞り込む前提。

自動スカウト設定案:
  上記条件をベースに自動スカウト機能を設定することも可能。
  自動スカウトの場合、テンプレートメッセージが自動送信される。
```

### 絞り込み条件

出力フォーマット：
```
■ Recruit Direct Scout 絞り込み条件

フリーワード: ◯◯（具体的なスキル・経験キーワード）
業界: ◯◯
職種: ◯◯
年収帯: ◯◯万円以上
勤務地: 東京都
経験年数: ◯年以上
レジュメ更新日: 直近1ヶ月以内（推奨）

想定ヒット感: 少なめだが精度高い。
```

### 検索条件のポイント
- RDS独自の業界・職種カテゴリ名を使う（例：業界は「IT・通信」「コンサルティング」等、職種は「経営企画・事業企画」「Webエンジニア」等）
- フリーワードは日本語中心
- 年収帯フィルタが有効（ハイクラス層が多いため、年収500万以上等で絞ると精度UP）
- レジュメ更新日が古い候補者は転職活動が終了している可能性が高い

## スカウト文面の生成ルール

### Recruit Direct Scout向けの特性

| 項目 | ルール |
|------|-------|
| 日程調整URL | **含めない**（BAN対策） |
| トーン | 丁寧・ビジネス寄り。条件面を具体的に |
| 長さ | **400〜600文字** |
| フォーマット | プレーンテキスト |
| 言語 | 日本語 |

### 文面構成のポイント

RDSの利用者は転職意欲が比較的高い。「今すぐではないが良い話があれば」という層も含めて、条件面の具体性が判断材料になる。

効果的なアプローチ：
1. 丁寧な自己紹介と連絡の経緯
2. 着目ポイント（候補者の経歴への評価）
3. ポジションの魅力（ビジネスの面白さ + 条件面）
4. **年収レンジ、勤務地、リモート可否を明記**（これがRDSでは特に重要）
5. 選考フローがあれば簡潔に触れる
6. 返信を促す形で締める（URLなし）

### テンプレート

```
{候補者名}様

はじめまして。
合同会社Plus X代表の藤本と申します。
弊社は興味深いポジション・ビジネスを有している企業の紹介に強みを持つヘッドハンティング会社です。

{候補者名}様の御経歴を拝見し、ぜひともご紹介させていただきたいポジションがございます。

【着目させていただいたポイント】
{着目ポイント}

上記の御経験から、本ポジションにおいて大きな成果を発揮いただけると考えております。

【ポジションの概要と魅力】
◎（事業・ビジネスの面白さを企業名を伏せて記載）
◎（ポジションの役割・裁量の大きさを記載）
◎（成長環境・キャリアパスの観点で記載）

【条件面】
年収レンジ：◯◯万円〜◯◯万円
勤務地：東京都
リモートワーク：◯◯（フルリモート可 / 週◯日出社 等）
雇用形態：正社員
SO制度：◯◯（あれば記載）

もし少しでも御興味をお持ちいただけましたら、求人の詳細をご案内するお時間を頂戴できないでしょうか。
ご返信にてご都合のよい日時をお知らせいただけましたら幸いです。
（所要時間：60分／オンライン）

御検討のほど、何卒よろしくお願いいたします。
```

### 絶対に守ること
- **URLは絶対に含めない**（BAN対策）
- 条件面（年収・勤務地・リモート）は必ず明記する
- 企業名・プロダクト名は絶対に含めない
- 600文字を超えない
- 資金調達情報を記載する場合は流動的に表現

## ブラウザ操作ワークフロー（Claude in Chrome）

Recruit Direct Scoutでの候補者リストアップをブラウザ上で行う場合の手順。
Claude in Chrome のツールを使って以下の順序で操作する。

**URL：** https://ats.hrtech.rikunabi.com/client/scouting/scoutroom/list

### 手順（重要：検索の前にスカウトルームを作成する）

1. **スカウトルーム一覧画面を開く**
   - 上記URLにアクセスする

2. **「スカウトルームを作成」ボタンを押す**
   - スカウトルーム一覧画面にある「スカウトルームを作成」ボタンをクリック
   - **スカウトルーム名は求人のポジション名**で作成する（例：「データエンジニア（リード）」）
   - 必要な求人情報を入力し、求人票を作成する

3. **作成したスカウトルームから検索をスタートする**
   - スカウトルームが作成されたら、そのルーム内から候補者検索に進む
   - フリーワード、業界、職種、年収帯などのフィルタを、このリファレンスで生成した検索条件に基づいて設定する
   - まず広め条件で母集団を確認し、多すぎれば絞り込み条件に切り替える

4. **候補者をスカウトルーム内でリストアップする**
   - 検索結果の候補者一覧から、マッチ度が高そうな候補者を選定する
   - 評価フレームワーク（マッチ度4以上）を参考に、スカウトを送る価値がある候補者をピックアップ
   - スカウトルーム内で候補者を管理し、後からスカウトメッセージを送信する
