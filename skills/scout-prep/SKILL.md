# Scout Prep — 求人票からスカウト準備を一括生成

## 前提：合同会社Plus Xについて

このスキルは、ヘッドハンティング会社「合同会社Plus X」（代表：藤本）のスカウト業務で使用する。自社採用のダイレクトスカウトではなく、クライアント企業のポジションを候補者に紹介する立場でのスカウトである。

この前提から導かれる重要なルール：
- スカウト文面に**クライアント企業名・プロダクト名を記載しない**（企業が特定できる情報は一切含めない）
- **勤務地は「東京都」レベルまで**に留める（区や住所は記載しない）
- **年収レンジとSO制度**は記載してOK
- 送信者は「合同会社Plus X 代表 藤本」として送る
- ポジションの魅力は伝えつつ、企業名は伏せたまま興味を引く構成にする
- **すべての媒体で日本語**のテンプレートを作成する（日本語話者に送信する前提）

## 処理の全体フロー

1. **求人票を解析する** — 職種、必須/歓迎スキル、業界、年収帯、勤務地、企業の魅力ポイントを構造化抽出
2. **ターゲットペルソナを定義する** — 理想的な候補者像を具体化（現職企業イメージ、経験年数、保有スキルなど）
3. **各媒体向け検索条件を生成する** — 広め条件と絞り込み条件の2パターン
4. **候補者評価フレームワークを用意する** — マッチ度5段階評価と着目ポイントの基準
5. **スカウトメール文面を生成する** — 媒体ごとにトーン・長さ・URL有無を調整した文面

## Step 1：求人票の解析

求人票から以下の情報を抽出し、構造化する。抜け漏れがあっても、求人票に書かれている範囲で最善を尽くす。

### 抽出項目

- **企業名**（内部管理用。スカウト文面には記載しない）
- **ポジション名**（正式名称）
- **募集背景**（なぜ今このポジションが必要なのか）
- **業務内容**（主要な3〜5つの業務）
- **必須スキル・経験**（箇条書きで）
- **歓迎スキル・経験**（箇条書きで）
- **求める人物像**（ソフトスキル・カルチャーフィット）
- **年収レンジ**
- **勤務地・リモート可否**
- **雇用形態**
- **企業の魅力ポイント**（候補者に響きそうな訴求ポイントを3〜5つ。企業名を伏せた状態で伝えられる形に翻訳する）
- **技術スタック / 開発環境**（該当する場合）
- **選考フロー**（該当する場合）

## Step 2：ターゲットペルソナの定義

求人票の情報をもとに、「実際にどういう人を探すのか」を具体化する。ここが曖昧だと検索条件もメール文面もぼやけるので、できるだけリアルな候補者像を描く。

### ペルソナに含める要素

- **現在の想定職種**（どんな肩書の人か）
- **現在の想定企業群**（具体的な企業名を5〜10社挙げる。同業界・同規模・類似課題を持つ企業）
- **経験年数の目安**
- **保有していそうなスキルセット**
- **転職動機として響きそうなポイント**（現職で感じていそうな課題やキャリアの次のステップ）

ペルソナは「メインターゲット」と「サブターゲット」の2パターン作る。

## Step 3：各媒体向け検索条件の生成

4つの媒体それぞれについて、**広め条件**と**絞り込み条件**の2パターンを生成する。

### 媒体別ガイドライン

#### LinkedIn
- 検索はキーワードベースが中心。Boolean検索（AND / OR / NOT / 引用符）が使える
- 広め条件：業界 + 職種キーワード（日英両方）で幅広く
- 絞り込み条件：スキルキーワードを追加、現在の会社名や過去の会社名で絞る
- プロフィールが英語の場合もあるため、**日本語と英語の両方のキーワード**を検索条件に含める
- 出力例：`キーワード: "data governance" OR "データガバナンス" AND ("manager" OR "lead" OR "マネージャー")`
- ただし**スカウト文面は日本語**で作成する（検索キーワードのみ日英併記）

#### RecruitDirectスカウト
- フリーワード検索 + 業界・職種などのカテゴリ選択式
- 広め条件：フリーワードは広めに、業界カテゴリで絞る
- 絞り込み条件：フリーワードを詳細に、年収帯・経験年数で絞る
- 出力にはフリーワードとカテゴリ選択の両方を含める

#### OpenWork
- 職種・業界・年収帯などの条件検索が中心
- 口コミデータがあるのが特徴なので、「企業への満足度」なども検索に使える場合がある
- 広め条件：職種カテゴリ + 業界で
- 絞り込み条件：スキルキーワード + 年収帯で

#### U-Trust
- 人材データベース検索。スキルタグやフリーワードが中心
- 広め条件：大きめのスキルカテゴリで
- 絞り込み条件：具体的なスキルタグ + 経験年数で

### 出力フォーマット（各媒体共通）

```
### [媒体名]

#### 広め条件（まず多くの候補を見たいとき）
- フリーワード / キーワード：〇〇
- 業界：〇〇
- 職種：〇〇
- その他フィルタ：〇〇
- 想定ヒット感：多め。ここから絞り込む前提

#### 絞り込み条件（ピンポイントで探したいとき）
- フリーワード / キーワード：〇〇
- 業界：〇〇
- 職種：〇〇
- 年収帯：〇〇
- 経験年数：〇〇
- その他フィルタ：〇〇
- 想定ヒット感：少なめだが精度高い
```

## Step 4：候補者評価フレームワーク

候補者を見つけた後、スカウトを送るかどうかの判断基準を用意する。このフレームワークは、リサーチ段階で候補者の職務経歴を確認する際に使う。

### マッチ度評価（5段階）

| スコア | 基準 |
|--------|------|
| 5 | 必須スキルをすべて満たし、歓迎スキルも複数該当。即戦力として高い確信がある |
| 4 | 必須スキルをほぼ満たし、歓迎スキルの一部も該当。十分にマッチする |
| 3 | 必須スキルの大半は満たすが、一部に不足あり。ポテンシャル採用の可能性 |
| 2 | 必須スキルの一部のみ該当。マッチ度は低いがキャリアチェンジ前提なら可能性あり |
| 1 | 求人要件とほぼ合致しない |

### 着目ポイントの作成ルール

マッチ度が**4以上**の候補者に対してのみ、着目ポイントを作成する。

着目ポイントのルール：
- **3点程度**を箇条書きで作成する
- 各ポイントは「◎」で始める
- 候補者の**強み・市場価値を褒める表現**を必ず入れる
- 仮説（推測に基づく部分）は「**〜ではないかとお見受けしています**」と明記する
- 求人の必須スキルや歓迎スキルとの具体的な接点を示す

着目ポイントの例：
```
◎〇〇業界にて約10年にわたりデータマネジメント領域で経験を積まれており、特に全社的なデータガバナンス体制の構築をリードされたご実績は非常に希少性が高いと考えております。
◎ISMSの導入・運用にも携わられており、情報セキュリティの観点からもデータの取り扱いに精通されているのではないかとお見受けしています。
◎直近では組織横断のプロジェクトマネジメントもご経験されており、本ポジションで求められるステークホルダーとの連携力をお持ちではないかとお見受けしています。
```

## Step 5：スカウトメール文面の生成

### 絶対に守るルール

1. **クライアント企業名・プロダクト名を絶対に記載しない**。業界や事業の特徴は伝えてよいが、企業が特定できる固有名詞は含めない
5. **資金調達情報は流動的に表現する**。具体的な金額・時期は企業特定につながるため、「100億円以上の大型調達を昨年実施済み」のように幅を持たせた表現にする。「〇〇億円のシリーズD」のような具体的な記載は避ける
6. **業界の表現**：「医療系」よりも「ヘルスケア系」「ヘルスケアテック」のような表現を優先する（より広い印象を与え、企業特定リスクを下げるため）
2. 送信者は「合同会社Plus X 代表 藤本」
3. `{候補者名}` のようなプレースホルダーを使い、パーソナライズすべき箇所を明示する
4. 着目ポイント `{着目ポイント}` はプレースホルダーとして記載し、実際のリサーチ時に候補者ごとに作成する

### 媒体別のURL・トーンルール

| 媒体 | 日程調整URL | トーン | 長さ目安 |
|------|------------|--------|----------|
| LinkedIn | 含める（SpirInc URL） | カジュアル・フラット。堅すぎると無視される。LinkedInの文化に合わせ、端的で人間味のある文面にする。**日本語で作成** | 200〜350文字 |
| U-Trust | 含める（SpirInc URL） | **チャット形式**で送信するため、やや短め・カジュアル寄り。IT/エンジニア系に強い媒体なので、技術的な面白さ・成長機会を訴求しつつ、チャットの自然なテンポを意識する | 250〜400文字 |
| OpenWork | **含めない**（BAN対策） | やや丁寧・共感ベース。口コミ文化のプラットフォームなので、「働きがい」「カルチャー」に触れると響く | 300〜500文字 |
| RecruitDirectスカウト | **含めない**（BAN対策） | 丁寧・ビジネス寄り。転職顕在層が多い。ポジションの魅力と条件を具体的に伝える | 400〜600文字 |

### 媒体別の調整ポイント

#### LinkedIn向け
- **日本語**で作成する（日本語話者への送信を前提）
- LinkedInのメッセージはフィードの中で埋もれやすいので、**冒頭の1〜2行で興味を引く**ことが特に重要
- 「はじめまして」から入るよりも、候補者のプロフィールに触れた一文から入る方が開封率が高い
- 全体をコンパクトにまとめる（長いと読まれない）
- LinkedInに**直接ペーストできる形式**で出力する（Markdownの記法を使わない、プレーンテキストで）

#### U-Trust向け
- **チャット形式**で送信する媒体なので、メールほど長くしない
- 堅すぎない自然なトーンにする。ただしビジネスの最低限の丁寧さは保つ。**初対面のため「お疲れ様です」は使わない**（知り合い同士の表現であるため）。「はじめまして」や「初めてご連絡いたします」のように初対面として自然な入り方にする
- 技術的な具体性を出す。「データ基盤」「ガバナンス」など、職種に合った専門用語を使う
- IT人材が多い媒体なので、技術的なチャレンジや成長機会に触れると興味を引きやすい
- チャット特有の読みやすさを意識し、段落を短くする

#### OpenWork向け
- 「働きがい」「成長環境」「裁量」など、OpenWorkの口コミ文化に馴染むキーワードを使う
- URLは含めず、返信を促す形にする

#### RecruitDirectスカウト向け
- 転職意欲が比較的高い層が多いので、条件面（年収レンジ、リモート可否）を具体的に示す
- URLは含めず、返信を促す形にする
